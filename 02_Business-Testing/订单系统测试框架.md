# 订单系统功能测试框架

> **任何功能 = 场景 + 输入 + 输出 + 角色 + 状态 + 异常**

------

#  一、业务场景分析（Scenario）

> **目的：确保不会漏掉用户真实的使用路径**

问自己：

- 用户通常会怎么用这个功能？
- 用户会从哪些入口触发？
- 使用流程有几个步骤？

输出：

+ 业务流程图
+ 所有入口场景
+ 用户使用路径

------

# 二、输入分析（Input）

> **目的：确保输入导致的所有分支逻辑都被覆盖**

输入包括：

### 1. 正常输入

- 符合规则的输入

### 2. 非法输入

- 空输入
- 超长输入
- 格式不对
- 特殊字符、emoji、敏感词等

### 3. 边界输入

- 最大值/最小值
- 库存为 0
- 数字溢出
- 列表第一项、最后一项

输出：

+ 输入清单
+ 每个输入的预期行为

------

# 三、输出分析（Output）

> **目的：验证“结果是否完全正确”**

输出包括：

### 1. UI 输出

- 显示是否更新
- 金额是否正确
- 列表是否刷新

### 2. 后端输出

- 数据库是否写入
- 关联数据是否生成（订单、评论记录、日志）

### 3. 业务输出

- 订单状态是否正确
- 库存是否扣除
- 评论点赞数量是否正确

输出：

+ UI 层验证点
+  数据层验证点
+ 业务结果验证点

------

# 四、角色分析（Role）

> **目的：每种用户权限不同 → 触发的逻辑不同**

典型角色：

- 普通用户
- 服务人员
- 管理员
- 未登录用户
- 黑名单用户

你要做一个矩阵：

> **角色 × 功能 = 所有组合都必须测试**

输出：
 ✔ 角色权限矩阵
 ✔ 每个角色允许/禁止的操作

------

# 五、状态分析（State）

> **目的：任何状态变化都需要测试，不然必漏点**

功能相关状态包括：

### 1. 业务数据状态

- 商品（有库存/无库存/下架/限购）
- 动态（正常/被删除/审核失败）
- 优惠券（已领/未领/过期/已用）
- 服务类型 
  - 订单（待支付、待接单、待服务、服务中、待评价、已完成、已取消）
  - 支付（未支付、支付中、已退款、已支付）
  - 售后（待处理、已完成、已拒绝、已取消）

- 商城
  - 订单（待支付、待发货、待收货、待评价、已完成、已取消）
  - 支付状态（未支付、支付中、已支付、已退款）
  - 售后（待处理、已取消、待退货、待收货、待确认、已完成、已拒绝）


### 2. 用户状态

- 黑名单
- 未实名
- 未登录

### 3. 系统状态

- 自提点关闭
- 配送不可用
- 评论关闭
- 商品不能购买

做矩阵：

> **状态 × 操作 = 全覆盖**

输出：
 ✔ 状态矩阵
 ✔ 各状态下的期望行为

------

# 六、异常分析（Exception）

> **目的：系统异常是线上最常见问题，需要主动测**

异常包括：

### 1. 网络异常

- 提交时断网
- 弱网、超时

### 2. 接口异常

- 401（未登录）
- 403（权限不足）
- 429（频率限制）
- 500（服务器错误）

### 3. 并发异常

- 同时点击多次
- 多人同时购买最后一件库存

### 4. 不可控异常

- 第三方支付失败
- 第三方服务超时

输出：
 ✔ 异常路径
 ✔ 恢复策略
 ✔ 重试是否安全

# 补充 订单系统测试重点

订单系统是电商系统的核心，主要从:**功能、性能、安全**3个维度来测试业务完整性和数据一致性。

**业务场景测试点：**

+ 核心正向流程的测试，也就是用户从**浏览->加购->下单->支付->发货->收货->完成**的全链路。
+ 订单的状态流转：待支付、已支付、待发货、已发货、已完成、待评价、已取消、已退款
+ 在不同的时间节点去申请售后

这里面高风险点是 钱、状态流转、库存、数据一致性，所以可以设计各种金额以及优惠券来测试支付，重复下单库存超卖，库存释放，状态是否可以从待支付变已发货等

**异常处理测试点：**

+ 网络与第三方故障:如支付回调丢失、物流接口超时。
+ 支付回调延迟
+ 边界值: 超大金额、超长地址、商品数量上限等。
+ 幂等性:防止用户重复提交订单、支付中心重复回调。
+ 弱网测试

**性能测试：**

+ 下单接口、支付回调接口
+ 抢优惠券接口
+ 数据库慢sql

**安全测试：**

权限测试:不能横向越权，只能看到自己的订单信息，不能纵向越权，看到管理员才能看的信息

关键字段 加密传输（订单金额去做修改）



**上线后**

**增加一个观察者意识**

+ 核心功能是否有异常反馈
+ 是否有用户 / 客服问题
+ 是否有异常数据
