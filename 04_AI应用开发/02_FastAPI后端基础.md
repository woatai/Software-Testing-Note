# FastAPI 后端基础

FastAPI 是 AI 应用开发里很实用的后端框架，适合快速把 Python 能力封装成 HTTP 接口。

## 必会概念

- 路由：一个 URL 对应一个处理函数。
- 请求参数：query、path、body。
- 响应结构：返回 dict、list 或 Pydantic 模型。
- 状态码：成功、参数错误、服务异常。
- 启动服务：本地运行后用浏览器或 Apifox 调试接口。

## 学习顺序

1. 写一个 `/health` 接口，返回服务状态。
2. 写一个 `/summarize` 接口，接收文本并返回摘要。
3. 写一个 `/analyze-interview` 接口，接收面试转写文本并返回分析结果。
4. 给接口加参数校验和错误提示。
5. 用 Apifox 或 Postman 保存接口用例。

## 和测试经验的连接

- 接口文档、请求参数、响应断言，这些都和接口测试经验直接相通。
- 做 AI 应用时，也要关注异常输入、超长文本、空文本、重复提交和错误返回。
- 你已有的 Apifox、Postman、HTTP 笔记可以直接复用。
