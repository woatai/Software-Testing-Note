# AI 应用开发路线

目标不是算法训练岗，而是能把大模型能力接到真实业务流程里，做出可演示、可复盘、可迭代的小应用。

## 阶段一：补齐应用开发底座

- Python：函数、文件处理、异常、虚拟环境、依赖管理。
- HTTP：请求方法、状态码、请求体、响应体、鉴权。
- FastAPI：接口、参数、请求体、返回结构、错误处理。
- Git：分支、提交、回滚、README、项目说明。

## 阶段二：接入 LLM

- 调用模型 API，完成文本总结、分类、抽取、改写。
- 学会设计 Prompt：角色、任务、输入、输出格式、约束。
- 使用结构化输出，让模型返回 JSON 或固定 Markdown 模板。
- 处理失败：超时、空结果、格式错误、敏感信息。

## 阶段三：做 AI 应用项目

- 文档问答：本地笔记或面试资料问答。
- 面试分析：输入转写文本，输出薄弱点和复习计划。
- 测试提效：根据需求生成测试点、用例、接口检查清单。
- 简历优化：根据岗位 JD 调整项目表达。

## 项目练习方向

项目先不固定，等确定你真正想练的 AI 应用方向后再建项目笔记。

选择项目时优先看三点：

- 能不能展示 Python 后端和接口能力。
- 能不能真实调用 LLM API，而不是只写概念。
- 能不能沉淀成简历里讲得清楚的小作品。
